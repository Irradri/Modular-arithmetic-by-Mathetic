Modular Arithmetic

Modular arithmetic is a system of arithmetic for integers, which considers the remainder. The modern approach to modular arithmetic was developed by Carl Friedrich Gauss in his book Disquisitiones Arithmeticae, published in 1801. It leads to solutions to some mathematical problems in real life and generally appears in fields like cryptography, computer science, and computer algebra. For Instance:

What is the remainder when 72015 is divided by 25?

What is the remainder when the sum of the digits of the sum of the digits of the sum of the digits of 77777777 is divided by 9?

Some fancy notation

First things first, Modular arithmetic is a game of integers. You can't involve fractions or irrationals or complex numbers here.

A number x mod N is the equivalent of asking for the remainder of x when divided by N.
For example, when we divide 23 by 7, we get remainder 2. So 23 mod 7 = 2
Similiarly, we can show that 54 mod 8 = 6, 45 mod 5 = 0 and so on...

Another way of saying it is that if x = Nq+r [N is divisor, q is quotient and r is remainder] then x mod N = r.

If integers a and b have the same remainder upon division by N, we say that a ≡ b (mod N).

Or Simply, a ≡ b (mod N) means the same as a mod N = b mod N

(notice that it's ≡ not = | ≡ means 'congruent' like = means 'equals')

So, 52 ≡ 24 (mod 7) , 36 ≡ 10(mod 13) and so on...

Another way of defining this is that if a ≡ b (mod N) then (a - b) is an integer multiple of N. How? if a mod N = b mod N, then a = Np + r and b = Nq + r,
so (a - b) = N(p - q) = integer multiple of N.

In the above examples, 52 - 24 = 28 = 7 × 4 and 36 - 10 = 26 = 13 × 2. Cool!

Properties of addition

  1. a mod N + b mod N ≡ a + b (mod N).
    a = Np + r1 and b = Nq + r2. So a + b = N(p + q) + r1 + r2. So a + b ≡ r1 + r2 ≡ a mod N + b mod N(mod N)
  2. If a ≡ b(mod N), then a + k ≡ b + k(mod N) for any integer k.
    If a ≡ b(mod N), then (a - b) divides N. Now, (a - b) = ((a + k) - (b + k)). So, (a + k) - (b + k) divides N => a + k ≡ b + k (mod N)
  3. If a ≡ b(mod N) and c ≡ d(mod N), then a + c ≡ b + d(mod N).
    a ≡ b(mod N) and c ≡ d(mod N) => (a - b) and (c - d) divides N => (a - b) + (c - d) divides N => (a + c) - (b + d) divides N => a + c ≡ b + d(mod N)
  4. If a ≡ b(mod N), a ≡ b + kN(mod N) for any integer k
    a ≡ b(mod N) and 0 ≡ kN(mod N) => a + 0 ≡ b + kN(mod N) => a ≡ b + kN(mod N)

Examples:

Q: Find the remainder of 313 + 841 when divided by 24.

Solution 1:(313 + 841) ≡ 1154 ≡ 2(mod 24)

Solution 2:313 ≡ 1 and 841 ≡ 1 (mod 24)
∴ 313 + 841 ≡ 1 + 1 ≡ 2(mod 24)

∴ (313 + 841)mod 24 = 2 mod 24 = 2

Q: Find the remainder when 123 + 234 + 32 + 56 + 22 + 12 + 78 is divided by 3

We know that 123 ≡ 0(mod 3), 234 ≡ 0(mod 3), 32 ≡ 2(mod 3), 56 ≡ 2(mod 3), 22 ≡ 1(mod 3), 12 ≡ 0(mod 3), and 78 ≡ 0(mod 3).

This implies that: 123 + 234 + 32 + 56 + 22 + 12 + 78 ≡ 0 + 0 + 2 + 2 + 1 + 0 + 0 ≡ 5 ≡ 2(mod 3)

and thus the answer is 2 mod 3 = 2

Note: if 0 ≤ b < N then b mod N = b [think about it if you don't get why]

Properties of multiplication

  1. a mod N × b mod N ≡ ab (mod N).
    a = Np + r1 and b = Nq + r2. So ab = N2pq + Nqr1 + Npr2 + r1r2 = N(Npq + qr1 + pr2) + r1r2 => ab ≡ r1r2 ≡ a mod N × b mod N(mod N)
  2. If a ≡ b(mod N), then ak ≡ bk(mod N) for any integer k.
    a ≡ b(mod N) => (a - b) divides N => k(a - b) divides N => (ak - bk) divides N => ak ≡ bk (mod N)
  3. If a ≡ b(mod N) and c ≡ d(mod N), then ac ≡ bd(mod N).
    a ≡ b(mod N) and c ≡ d(mod N) => a mod N = b mod N and c mod N = d mod N => a mod N × c mod N = b mod N × d mod N
    Now a mod N × c mod N ≡ ac and b mod N × d mod N ≡ bd(mod N)
    ∴ ac ≡ bd(mod N)

Examples:

Q: (8 × 16) mod 7 ?

Solution 1: (8 × 16) ≡ 128 ≡ 2 (mod 7)

Solution 2: 8 ≡ 1 and 16 ≡ 2 (mod 7) => 8 × 16 ≡ 1 × 2 ≡ 2 (mod 7) = 2

Q: Find the remainder when 124 × 134 × 23 × 49 × 235 × 13 is divided by 3

In this example, multiplying the numbers would be very tedious. Instead, we use property 3 repeatedly. We know that 124 ≡ 1(mod 3), 134 ≡ 2(mod 3),
23 ≡ 2(mod 3), 149 ≡ 1(mod 3), 235 ≡ 1(mod 3) and 13 ≡ 1(mod 3). Therefore,

124 × 134 × 23 × 49 × 235 × 13 ≡ 1 × 2 × 2 × 1 × 1 × 1 ≡ 4 ≡ 1(mod 3) = 1

Exponential Property

If a ≡ b(mod N) then ak ≡ bk(mod N) for any non-negative integer k
a ≡ b(mod N) which implies
a ≡ b(mod N) which again implies
a ≡ b(mod N) [bruh]
.
.        [writing k times]
.
a ≡ b(mod N)
∴ by multiplying all these using multiplication property III, we get ak ≡ bk(mod N)

Note: k is non-negative as it doesn't make sense to write an equation negative number of times
Q: What is 416 mod 5?

Strategy:Find n such that 4n ≡ 1 or -1 (mod 5)
We can directly see that 4 ≡ -1 (mod 5)
So 416 ≡ (-1)16 ≡ 1 (mod 5)
∴ 416 mod 5 = 1

Amazing, right? We didn't have to compute 416, which is really large

Q: What is the last digit of 1717 ?

Finding the last digit means the same as applying mod 10.
If we try applying the strategy above, finding n is very difficult through trial and error.
Instead, we use this clever trick:
17 ≡ -3(mod 10) => 1717 ≡ (-3)17(mod 10)
Now, (-3)2 ≡ 9 ≡ -1(mod 10) => (-3)16 ≡ (-1)8 ≡ 1(mod 10) => (-3)17 ≡ -3 ≡ 7(mod 10)
∴ 1717 ≡ (-3)17 ≡ 7(mod 10)
∴ Last Digit = 7

Q: Find the last three digits of 240.

240 = (210)4 = (1024)4
(1024)4 ≡ (24)4 ≡ (576)2 ≡ 331776 ≡ 776 (mod 1000)
∴ 240 mod 1000 = 776
